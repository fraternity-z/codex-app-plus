# 测试计划

## 1. 测试目标与范围

基于已确认需求与 `plan.md`，本计划仅覆盖左侧工作区列表：
- 拖拽排序正确性
- 智能吸附分组规则（R1~R5）正确性
- 顺序状态持久化正确性
- 与现有交互（点击/展开/菜单）冲突处理
- 拖拽动画性能基线
- 相关回归范围

不包含：主会话区、HostBridge、Rust/Tauri 后端、线程列表拖拽。

## 2. 验收标准（Pass/Fail）

### 2.1 功能验收标准
1. 工作区项可通过鼠标拖拽稳定改变顺序，放置后顺序与可视结果一致。
2. 智能吸附规则满足计划定义：
   - R1：0.35/0.65 区间前插/后插判定正确；
   - R2：中心区悬停 >=180ms 触发自动插队；
   - R3：同组邻居中心线 16px 内优先吸附；
   - R4：落入同组连续片段时按上下方向归一化到片段边界；
   - R5：优先级与回退生效（R3 > R2 > R1，异常回退最近合法 index）。
3. 刷新应用后顺序保持不变（localStorage 持久化生效）。
4. 拖拽取消不应写入新顺序。
5. 选中态、展开态、菜单操作不因拖拽引入异常。

### 2.2 性能验收标准（基线）
1. 在 30 个工作区项场景下，连续拖拽 10 秒，主观无明显卡顿（无持续性“停顿 > 300ms”）。
2. DevTools Performance 观察到拖拽主路径以 `transform` 位移为主，不出现持续密集的全列表强制布局抖动。
3. 拖拽中无明显动画撕裂、占位错位、插入指示线闪烁。

### 2.3 通过/阻塞判定
- **通过（Go）**：P0 用例 100% 通过，P1 通过率 >= 90%，无 P0/P1 阻塞缺陷。
- **阻塞（Block）**：任一 P0 失败；或出现数据错乱/顺序丢失/交互不可用/严重性能退化。

## 3. 测试环境要求

- 系统：Windows 10/11（桌面鼠标交互）
- 应用：当前实现分支构建版本
- 浏览器内核环境：与应用内 WebView 一致
- 数据准备：至少 8 个工作区项，需包含可形成同组与异组的路径样本
  - 同组样本示例：`E:/code/proj-a`、`E:/code/proj-b`
  - 异组样本示例：`E:/work/demo-x`、`D:/lab/demo-y`
- 性能观测工具：DevTools Performance（仅记录，不执行自动化基准）

## 5. 测试用例

> 说明：以下用例为可执行步骤；每条均给出输入动作与期望结果。优先级分级：P0（必须）、P1（高）、P2（中）。

| 用例ID | 优先级 | 覆盖项 | 输入动作（可执行步骤） | 期望结果 |
|---|---|---|---|---|
| TC-FUNC-001 | P0 | 功能正确性/基础重排 | 在左侧列表拖拽第 1 项到第 4 项之后并释放 | 列表顺序立即更新为目标顺序，拖拽项落位正确 |
| TC-FUNC-002 | P0 | 功能正确性/多次重排 | 连续执行 3 次不同方向拖拽（上移、下移、跨段） | 每次结果均与放置位置一致，无错位/回弹错误 |
| TC-SNAP-001 | P0 | R1 吸附规则 | 将拖拽指针落在目标项上方 35% 区域释放 | 判定为前插，最终 index 为目标项前一位 |
| TC-SNAP-002 | P0 | R1 吸附规则 | 将拖拽指针落在目标项下方 65% 区域释放 | 判定为后插，最终 index 为目标项后一位 |
| TC-SNAP-003 | P0 | R2 中心悬停插队 | 指针停留在目标中心区（35%~65%）并保持 >=180ms 后释放 | 触发自动插队：按相对方向插入目标前/后 |
| TC-SNAP-004 | P0 | R3 同组吸附 | 拖拽 `groupKey=code` 项靠近同组邻居中心线 <=16px 释放 | 优先吸附到同组邻居侧，而非异组线性位置 |
| TC-SNAP-005 | P1 | R4 组内紧凑排队 | 将同组项从片段外拖入同组连续片段边缘（上方进入/下方进入各一次） | 上方进入落在片段头部；下方进入落在片段尾部后 |
| TC-SNAP-006 | P1 | R5 优先级 | 构造同时满足 R1/R2/R3 的位置并释放 | 最终结果遵循 R3 优先；若异常则回退合法 index |
| TC-PERSIST-001 | P0 | 状态持久化 | 执行一次重排后刷新应用 | 刷新后顺序保持与刷新前一致 |
| TC-PERSIST-002 | P1 | 取消不持久化 | 开始拖拽后按 Esc 取消或拖出容器取消 | 顺序恢复原始值，localStorage 不写入新顺序 |
| TC-INTERACT-001 | P0 | 交互冲突/点击 | 对工作区项进行短位移点击（低于拖拽阈值） | 仅触发点击/选中，不触发排序 |
| TC-INTERACT-002 | P1 | 交互冲突/展开收起 | 展开/收起后立即拖拽同项并放置 | 拖拽可用，展开态不异常、不串位 |
| TC-INTERACT-003 | P1 | 交互冲突/菜单 | 打开项菜单后关闭，再执行拖拽 | 菜单与拖拽互不干扰，无误触发 |
| TC-STATE-001 | P0 | 状态一致性/选中态 | 选中某项后将其拖拽到新位置 | selectedRootId 仍指向该项，选中视觉不丢失 |
| TC-STATE-002 | P1 | 状态一致性/展开态 | 多项展开状态下执行跨组拖拽 | 各项展开态保持原语义，不因 index 变化错绑 |
| TC-PERF-001 | P0 | 性能基线/流畅度 | 30 项列表中连续拖拽 10 秒并观察 | 无持续明显卡顿，交互可连续完成 |
| TC-PERF-002 | P1 | 性能基线/渲染路径 | 使用 DevTools 录制拖拽过程 | 主路径以 transform 为主，无持续高频强制布局热点 |
| TC-REG-001 | P1 | 回归/新增删除兼容 | 重排后新增一个工作区，再删除另一个 | 新增/删除行为正常，最终顺序与预期一致 |
| TC-REG-002 | P1 | 回归/切换工作区与会话 | 重排后切换不同工作区并打开会话列表 | 导航与会话展示正常，无串数据 |
| TC-REG-003 | P2 | 边界/单项与空列表 | 空列表、单项列表分别尝试拖拽 | 不报错；无无意义重排写入 |

## 6. 回归范围

1. 工作区相关：选择、展开/收起、右键菜单、新增、删除。
2. Home 侧边栏链路：`HomeScreen -> HomeView -> HomeSidebar -> WorkspaceSidebarSection` 参数透传。
3. 持久化链路：`useWorkspaceRoots` 与 localStorage 读写协议兼容性。
4. 非目标区域不受影响：线程列表、主会话区、设置页、终端页基础可用性抽检。

## 7. 缺陷分级与处理

- S1（阻塞）：无法完成拖拽排序、顺序丢失、数据错乱、严重性能不可用。
- S2（高）：吸附规则错误导致高概率错误落位，或核心交互冲突明显。
- S3（中）：偶发视觉问题、不影响结果正确性。
- S4（低）：文案/样式轻微偏差。

处理原则：S1/S2 必须修复并回归通过后方可进入测试阶段通过结论。

## 8. Test Stage Definition of Done（通过/阻塞准入标准）

### 8.1 通过准入（DoD-Pass）
1. 所有 P0 用例通过。
2. P1 用例通过率不低于 90%，且无未接受的 S1/S2 缺陷。
3. 性能基线用例（TC-PERF-001）通过，且 TC-PERF-002 无“持续严重布局抖动”结论。
4. 持久化与交互冲突关键路径（TC-PERSIST-001、TC-INTERACT-001、TC-STATE-001）全部通过。
5. 已形成可追溯测试记录（用例结果、缺陷单、回归记录）。

### 8.2 阻塞准入（DoD-Block）
满足任一条件即阻塞：
1. 任一 P0 用例失败。
2. 出现 S1 缺陷未修复。
3. 出现 S2 缺陷且影响主路径（拖拽排序/吸附/持久化/核心交互）。
4. 性能基线不达标并影响可用性。

## 9. 覆盖率要求

- 功能覆盖率：本计划列出的 6 大覆盖域（功能正确性、吸附规则、状态持久化、交互冲突、性能基线、回归范围）均需有通过记录。
- 自动化覆盖目标（实现阶段建议）：
  - 与拖拽规则计算相关的单元测试覆盖核心分支（R1~R5）
  - `useWorkspaceRoots` 重排与持久化路径覆盖
  - 目标代码覆盖率建议 > 80%（以模块级为准）
