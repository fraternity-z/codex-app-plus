## 1. 概述

### 背景
当前左侧工作区列表仅支持线性展示、选择与基础管理，不支持拖拽重排。已确认需求要求在**仅限左侧工作区列表**范围内实现工作区文件夹拖拽排序，并引入“自动排队插队”的智能吸附分组规则，同时保证拖拽过程具备高性能动画体验。

### 目标
1. 支持工作区文件夹在左侧列表中拖拽。
2. 拖拽完成后可稳定改变列表顺序并持久化。
3. 提供可执行、可预期的“自动排队插队”智能吸附分组规则。
4. 拖拽过程动画流畅，主路径以 `transform` 驱动，避免明显掉帧。

### 范围
- **包含**：`WorkspaceSidebarSection` 列表拖拽交互、排序计算、智能吸附规则、顺序持久化、相关样式与交互状态。
- **不包含**：主会话区、HostBridge API、Rust/Tauri 后端、其他列表（如线程列表）拖拽能力。

## 2. 需求分析

### 已确认需求拆解
1. 左侧工作区列表支持工作区文件夹拖拽。
2. 拖拽可改变排列顺序。
3. 支持“自动排队插队”的智能吸附分组规则。
4. 拖拽过程高性能动画。
5. 影响范围仅左侧工作区列表。
6. 按完整 5 阶段流程推进。

### 功能性要求
- 支持鼠标拖拽排序（以桌面场景为主）。
- 拖拽中实时显示目标插入位置与受影响项位移动画。
- 放置后更新 `roots` 顺序并写回现有 localStorage。
- 不破坏已存在的选中状态、展开状态与菜单操作链路。

### 非功能性要求
- 动画流畅：拖拽主路径避免频繁触发 reflow/repaint 热点。
- 交互稳定：避免点击与拖拽误触发冲突。
- 规则可解释：智能吸附行为需可被明确描述与复现。

## 3. 设计方案

### 3.1 总体方案
采用 `dnd-kit` 在 `WorkspaceSidebarSection` 实现可排序拖拽层；在 `useWorkspaceRoots` 增加重排能力（如 `reorderRoots`），延续现有 localStorage 持久化机制。改动控制在左侧工作区链路内。

### 3.2 架构落点
- UI 层：`src/features/workspace/ui/WorkspaceSidebarSection.tsx`
- 状态层：`src/features/workspace/hooks/useWorkspaceRoots.ts`
- 透传层：`src/features/home/ui/HomeSidebar.tsx`、`HomeView.tsx`、`HomeScreen.tsx`
- 样式层：`src/styles/replica/replica-shell.css`、`src/styles/replica/replica-sidebar-tree.css`

### 3.3 智能吸附分组规则（可执行规则定义）
以下规则以“可计算、可复现”为准，不使用模糊描述：

#### 规则 R1：基础插入判定（自动排队）
- 输入：拖拽指针 `pointerY`、目标项边界 `targetRect`。
- 判定：
  - 若 `pointerY < targetRect.top + targetRect.height * 0.35`，则判定为“前插”。
  - 若 `pointerY > targetRect.top + targetRect.height * 0.65`，则判定为“后插”。
  - 中间区间 `[0.35, 0.65]` 进入 R2/R3 进一步判定。
- 输出：候选插入 index。

#### 规则 R2：中心悬停插队（自动插队）
- 触发条件：指针位于目标项中心区间 `[0.35, 0.65]` 且持续悬停 `>= 180ms`。
- 行为：
  - 若拖拽项原 index < 目标 index，则插入到目标后（index + 1）。
  - 若拖拽项原 index > 目标 index，则插入到目标前（index）。
- 目的：减少用户在微小位移场景中的反复拉扯。

#### 规则 R3：相邻组吸附（智能分组）
- 为每个工作区定义 `groupKey`：
  1) 优先使用路径首层目录名（忽略盘符）；
  2) 若无法解析，退化为路径主机段/项目根名。
- 当候选插入点前后任一邻居 `groupKey` 与拖拽项一致，且指针距该邻居中心线绝对值 `<= 16px` 时：
  - 优先吸附至该同组邻居一侧（根据上下方向决定前/后）。
- 当不存在同组邻居时，回退到 R1/R2 结果。

#### 规则 R4：组内紧凑排队
- 若放置位置落在同组连续区间边缘（同 `groupKey` 连续片段），最终 index 归一化到该连续片段最近边界：
  - 从上方进入片段，插入片段头部。
  - 从下方进入片段，插入片段尾部后。
- 目的：形成“自动排队”的组内紧凑布局，避免同组项被打散。

#### 规则 R5：规则优先级与回退
- 优先级：`R3 > R2 > R1`，放置归一化时再执行 `R4`。
- 任一规则计算异常（边界缺失、索引越界）时，回退至纯线性 `arrayMove` 的最近合法 index。

### 3.4 高性能动画策略
- 列表位移使用 `transform: translate3d(...)`，避免通过 `top/left` 改位。
- 拖拽占位与插入指示线采用独立层，减少整体重排。
- 仅在必要状态更新 React state；指针移动高频数据优先使用 dnd-kit 内部机制。
- 对非拖拽项启用短时 easing 过渡，拖拽激活项关闭无关 transition 以降低抖动。

### 3.5 兼容与边界
- 空列表/单项列表时禁用重排逻辑。
- 拖拽取消（Esc/离开容器）时回退原顺序，不写入持久化。
- 与点击/展开/菜单冲突通过拖拽启动阈值隔离（例如最小位移阈值）。

## 4. 执行模式

### 执行模式选择

**推荐模式**：单 Agent（single-agent）

**选择理由**：
- 需求边界明确且集中在单一功能域（左侧工作区列表）。
- 状态、组件、样式链路短，串行推进更易保持规则一致性。
- 智能吸附规则需要统一口径实现与调参，避免多 Agent 并行导致行为分叉。

## 5. 实现步骤（5 阶段流程）

### 阶段 1：需求固化与规则冻结
- 将智能吸附规则 R1~R5 固化为实现约束与验收基线。
- 明确仅改动左侧工作区列表相关文件，排除 HostBridge/Rust 范围。
- 输出规则参数初值（0.35/0.65、180ms、16px）作为第一版。

### 阶段 2：状态能力扩展
- 在 `useWorkspaceRoots` 增加重排接口（基于索引移动）。
- 保持既有 localStorage 读写协议不变，仅扩展顺序更新路径。
- 确保重排后 `selectedRootId` 与现有删除/新增逻辑兼容。

### 阶段 3：拖拽交互接入
- 在 `WorkspaceSidebarSection` 引入可排序拖拽上下文。
- 为 workspace root 行接入 sortable item 能力与拖拽事件处理。
- 在拖拽结束时应用 R1~R5 计算最终 index 并调用重排接口。

### 阶段 4：智能吸附与动画优化
- 实装同组吸附、中心悬停插队、组内紧凑排队规则。
- 增加插入指示线、占位态、拖拽激活态视觉反馈。
- 优化动画路径为 transform 主导，控制渲染抖动与不必要重排。

### 阶段 5：收口与文档闭环
- 对照需求逐项核对：可拖拽、可排序、智能吸附、生效范围、性能策略。
- 记录实现约束、参数与已知边界，沉淀到实现总结文档。
- 准备后续 test-plan 的输入信息（本计划不包含测试计划章节）。

## 6. 风险和依赖

### 主要风险
1. **交互冲突风险**：拖拽手势与点击、展开、菜单触发冲突。
2. **规则感知偏差**：智能吸附结果与用户心智预期不一致。
3. **性能风险**：长列表或快速拖动下出现掉帧。
4. **状态一致性风险**：重排后选中项、展开态映射异常。

### 风险缓解
- 通过拖拽启动阈值与事件隔离降低误触发。
- 规则优先级固定（R3>R2>R1）并保留回退路径。
- 以 transform 动画为主，减少布局抖动来源。
- 对重排前后状态映射设置显式保护与边界校验。

### 关键依赖
- 前端依赖：`dnd-kit`（排序与拖拽基础能力）。
- 现有持久化能力：`storageJson` 的读写链路。
- 既有组件链路：`HomeScreen` → `HomeView` → `HomeSidebar` → `WorkspaceSidebarSection`。
